Brute Force（暴力破解）

工具：Burp Suite pro、爆破字典、代理：搜狐浏览器+FoxyProxy标准版

【漏洞介绍】暴力破解是一种尝试在不知道密码的情况下，使用各种可能的密码组合来尝试访问系统的攻击方法。攻击者通常使用计算机程序来自动化执行这个过程，以尝试尽可能多的密码组合。这种攻击方法常常用于破解弱密码，例如常见的单词、数字或者简单的组合密码。

【防御】

  1. 强密码策略：实施强密码策略可以有效地防止暴力破解攻击。建议密码长度至少12个字符，包含大小写字母、数字和特殊字符。
  2. 账户锁定：设置账户锁定策略可以有效地防止暴力破解攻击。例如，在多次失败的登录尝试之后锁定用户账户，只有管理员可以解锁。
  3. 暂时延迟：为了防止攻击者快速地执行暴力破解，可以在失败的登录尝试之间引入延迟。这可以防止攻击者进行高速尝试。
  4. 双因素身份验证：使用双因素身份验证可以增加安全性，使得攻击者更难破解密码。例如，使用令牌、指纹识别等。
  5. IP限制：限制IP地址可以防止攻击者使用不同的计算机和网络进行尝试。例如，只允许特定的IP地址进行登录。
  6. 安全日志：记录安全日志可以帮助检测和识别暴力破解攻击。例如，记录登录尝试、失败的登录尝试等。

总之，防御暴力破解攻击需要综合使用多种技术和最佳实践，包括强密码策略、账户锁定、暂时延迟、双因素身份验证、IP限制和安全日志等。同时，还需要定期更新密码、使用不同的密码等。

Command Injection（命令注入）

【漏洞介绍】命令注入就是在需要输入数据的地方输入了恶意代码，而且系统并没有对其进行过滤或者其他处理导致恶意代码也被执行，最终导致数据泄露或者正常数据被破坏。有些应用中，可能允许操作者向后台操作系统发送命令并执行，例如测试同网站的连接（ping命令），这种功能一般通过如PHP中的exec、shell_exec和ASP中的wscript.shell等函数实现。命令注入漏洞指的是在这些功能中，由于对操作者的输入或返回的输出结果控制不严导致能够让恶意操作者利用管道符等将恶意系统命令拼接在正常命令后并执行。

【简单的指令】

【常见命令拼接符】

  • A|B 不管A命令成功与否，都会去执行b命令。
  • &：A&B 各自执行
  • ||：A||B 先执行A，如果为假，再执行B
  • && ：A&&B 先执行A，如果为真，再执行B
  • ；： A;B 先执行A后执行B

File Inclusion（文件注入）

【漏洞介绍】文件注入漏洞（File Inclusion）是一种安全漏洞，允许攻击者在应用程序中引用和执行未经授权的文件。出现文件注入漏洞的原因通常是应用程序在动态引用文件时，没有正确地验证用户提供的输入，导致攻击者可以通过操纵这些输入来执行恶意代码。

【原理】文件注入漏洞发生在应用程序使用用户输入来动态生成文件路径的情况下。攻击者可以在用户输入中注入恶意代码，例如通过在文件路径中添加特殊字符或使用目录遍历攻击，来引用应用程序未经授权的文件。一旦攻击者成功执行了恶意代码，他们可以访问敏感信息，如配置文件、数据库凭据等，并且还可以在应用程序中执行任意代码。

【防御】为了防御文件注入漏洞，应用程序开发人员可以采取以下措施：

  1. 永远不要直接使用用户输入的数据来生成文件路径。而是使用白名单来限制允许访问的文件路径。例如，只允许访问特定的目录或文件。
  2. 对用户输入进行严格的输入验证和过滤，确保输入数据不包含特殊字符或目录遍历攻击。
  3. 使用安全编程语言，如PHP，提供内置函数来检查文件路径。例如，使用realpath()函数可以将文件路径规范化，从而防止目录遍历攻击。
  4. 避免在应用程序中使用可变文件名。如果必须使用可变文件名，确保在生成文件名之前对其进行过滤和验证。
  5. 对于包含敏感信息的文件，应该将它们放在应用程序根目录之外，并在操作系统层面上使用访问控制列表（ACL）限制文件的访问。

Insecure CAPTCHA

​ （不安全的验证码）

【常见的攻击方法】

  1. 自动化程序攻击：攻击者可以使用自动化程序来识别并绕过不安全的验证码，从而实现自动化攻击。
  2. 暴力破解攻击：攻击者可以使用暴力破解技术来破解验证码，从而实现攻击。例如，攻击者可以使用字典攻击或暴力破解工具来尝试所有可能的字符组合。
  3. 人工智能攻击：攻击者可以使用人工智能技术来分析和识别验证码，从而绕过验证码的限制。例如，攻击者可以使用深度学习算法训练模型来自动化地识别验证码
  4. 绕过技术

【防御】

  • 难度提升：提高验证码的难度可以有效地防止自动化程序和暴力破解攻击。例如，增加字符数量、添加背景噪音、旋转字符等。
  • 多种类型：使用多种类型的验证码可以增加攻击者绕过验证码的难度。例如，文字验证码、数学问题验证码、图像验证码等。
  • 时间限制：设置验证码的有效期限可以防止攻击者使用破解的验证码进行攻击。例如，验证码只在用户提交前几分钟内有效。
  • 验证码随机性：使用随机生成的验证码可以防止攻击者使用已知的验证码进行攻击。例如，使用随机的字符和字体。
  • 安全存储：验证码数据应该被安全地存储，以防止攻击者获取验证码数据并进行破解攻击。

总之，防御不安全的验证码需要综合使用多种技术和最佳实践，包括提高难度、使用多种类型、设置时间限制、使用随机性和安全存储等。

SQL Injection（SQL注入）

【漏洞介绍】Web应用程序通常会将用户输入的数据作为SQL查询语句的一部分发送到数据库中。攻击者可以利用这一点，将恶意SQL代码注入到Web应用程序中，从而欺骗数据库执行恶意代码，执行任意操作。

【防御】为了防止SQL注入漏洞，Web开发人员应该采取以下预防措施：

  • 输入验证和过滤：Web应用程序应该对所有输入参数进行验证和过滤，以确保它们符合预期的数据类型和格式。
  • 使用参数化查询：Web应用程序应该使用参数化查询，而不是将用户输入的数据直接嵌入到SQL查询语句中
  • 最小特权原则：Web应用程序应该使用最小特权原则，只授予数据库用户所需的最低权限
  • 错误信息处理：Web应用程序应该避免向用户显示详细的错误信息，因为攻击者可以利用这些信息发现漏洞并进行进一步攻击。
  • 使用安全编码实践：Web开发人员应该熟悉安全编码实践，并在开发过程中考虑安全性。

综上所述，Web开发人员应该对用户输入数据进行严格的验证和过滤，并使用参数化查询，以防止SQL注入漏洞。此外，他们还应该遵循最小特权原则，并使用安全编码实践来确保Web应用程序的安全性

欧几里得算法与欧几里得扩展算法

欧几里得算法

gcd(a,b) :a和b的最大公因子（公约数），下面同记为(a,b)，a，b∈Z

1.定理：(a,b) = (b,a mod b)**

推导过程：

∵ a，b∈Z，

∴ 有a = kb + r , k,r∈Z ①

则 a mod b = r ②

联合①②： a mod b = a-kb ③

设g = gcd（a，b）

则：a%g = 0（a mod g=0），b%g = 0；即a%g = b%g

可以推出：(a - b)%g = 0 ④

（a、b都是g的倍数关系，所以a和b的差值也是最大公因子g的倍数关系）

结合③和④

推出： （a mod b）%g = 0

所以g是a，b，（a mod b）的公因子

∴ (a,b) = (b,a mod b)

2.欧几里得算法（辗转相除法）

第一步：（a0，b0）

第二步：（a1，b1），a1= b0, b1 = a0%b0

……

第n步：（an，bn），当bn = 0时，an就是（a，b）最大公约数

扩展欧几里得算法

1.求逆元原理

推导过程：

设（a，b）是a，b的最大公约数，则有a.x1 = k1(a,b), b.y1 = k2(a,b) ,k1,k2∈Z “.”是乘法

可以推导出：a.x1 + b.y1 = k3(a,b) ，k3 = k1+k2，k3∈Z

所以必然存在

​ ax + by = （a，b ）

当（a，b）=1时，a，b互质（又称为互为素数），得：

​ ax + by =1 （x是a的逆元，y是b的逆元）

所以有：a*X = bY+1 （正负号写不写无所谓，因为它对公式没什么意义，Y可正可负）

​ X = （bY+1）/a

​ Y = （aX+1）/a

2.扩展的欧几里得算法

由欧几里得算法得：

​ （a，b） = （b，a mod b） ①

​ ∵ ax + by = (a,b) ②

联合①②得：

​ b.x1 +（a mod b）.y1 = (a,b) ③

​ a mob b = a%b = a - (a/b).b ④ “/“是除法

联合③④

​ gcd = （a，b） = b.x1+[a - (a/b).b].y1

​ =b.x1 + a.y1 - (a/b).b.y1

​ =a.y1 + b.[x1 - (a/b).y1]

对比：(a,b) = ax + by

可以推导出：

​ x = y1

​ y = x1 - (a/b).y1

椭圆曲线加法定义

说明：如果线上3个点位于同一直线上，那么它们的和为O

  1. O为加法单元，椭圆上任意一个点P，都有P+O=P；它运算上类似于0，但它意义不是一个简单的点的定义，无穷远点也是O。

  2. 加法逆元定义：P=（x，y），逆元p’=（x，-y）；关于X轴对称。这是因为P1、P2的连线延长到无穷远时，得到的是椭圆曲线上的另外一点O，即P1、P2、O三点共线，所以P1+P2+O=O，P1+P2=O，即P2=-P1。（如图b曲线，当P趋向无穷远时，P1、P2、无穷远点这3个点就共线了）

  3. 如图a，设Q和R是椭圆上的x坐标不同的两点。Q+R的定义：画一条Q和R的直线，与曲线交于P1（这交点是唯一的，除非所做的直线是Q和R的切线，此时P1=Q=R），由于Q、R、P1这3点共线，则Q+R+P1=O，得Q+R=-P1。

  4. 点Q的倍数（倍乘）定义：在Q点做椭圆曲线的一条切线，设切线与椭圆曲线交于点S，则2Q=Q+Q=-S。

  5. 椭圆曲线加法符合的运算法则：交换律、结合律

RSA加密算法及其数字签名

1、密钥产生

  • 选取两个大素数p和q

  • 计算n=p*q，φ(n)=(p-1)(q-1)，即φ(n)是n的欧拉函数值

  • 选取随机数e，1<e<φ(n)，且 gcd(φ(n)，e)=1

  • 计算d：

    ​ d*e=1 mod(φ(n))

  • 公钥{e,n}，私钥{d,n}

说明：

​ a.gcd(φ(n)，e)=1：最大公因子/公约数为1，即e和φ(n)互素

​ b.d*e=1 mod(φ(n))：由逆元定义得知，e和d互为逆元

​ c.到2021年为止，n的有效安全二进制位数为2048位（十进制大概608-609位），但1024位的也还有在使用
