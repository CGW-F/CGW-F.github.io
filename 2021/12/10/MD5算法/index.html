MD5算法

一、算法描述

​ 参数说明：

​ L：分组数

​ N：消息总的字数

算法的输入为任意长，输出为128比特的消息摘要；算法过程一般分一下几个步骤：

  • 步骤1：消息填充。

    ​ 消息填充是必要的，消息填充后的比特长度len满足448 = n mod 512，即len模512的余数为448；留下64比特记录步骤2的消息长度（512-448=64）。填充规则：首位填充位为1，其余补0；要注意，即使消息长度恰好为448比特，也要进行填充（填充512位），所以填充位的长度为1~512位。

  • 步骤2：附加消息长度

    ​ 消息长度为64比特（步骤1留下的），其表示还没被填充前的真实消息长度。采用的是小端方式填写，如果消息长度大于，则对其进行取模，模数为。

    小端方式：按数据的最低有效位优先的顺序进行存储数据，从低地址位开始存，如最低有效位存在最低地址位。相反的存储方式为大段方式。

  • 步骤3：消息分组

    ​ 执行完步骤1,2，消息总长度为512的倍数（这里记为L倍），把消息分为L组，每组512比特，分组记为，每一个分组可以表示16个32比特的字（16 x 32 =512)，那么消息的总字数N=16L，若按字节来分组，则为

  • 步骤4：MD缓冲区初始化

    ​ 缓冲区的大小为128比特，用于存储中间结果和最终摘要。缓冲区可以表示为4个32比特长的寄存器（A、B、C、D），每个寄存器都是以小端方式存储数据，其初始值IV如下：

    ​ A = 67452301

    ​ B = EFCDAB89

    ​ C = 98BADCFE

    ​ D = 10325476

  • 步骤5：以分组为单位对消息进行处理

    ​ 每一个分组（q=0,1,2，…，L-1)经过压缩函数处理。是算法的核心，其包括4轮处理过程，每一轮经过16步迭代运算。每轮处理的输入是（，），是A，B，C，D 4个缓冲区当前值，其初始值是IV；其中，每一轮输出为，也是下一轮处理的输入。分组处理的具体结构在第二部分介绍。

  • 步骤6：输出

    步骤5的最后一个输出（)就是最终结果：128位的消息摘要。

​ 第一部分的步骤3~步骤6都可以归结为分组处理。

​ 的4轮处理过程和结构是一样的，只是所用的逻辑函数不同，逻辑函数有F、G、H、I。输入：（，)；（q=0,1,2，…，L-1)是当前的消息分组；是寄存器四个A、B、C、D的存储值，本轮输出的也是下一轮运算的输入，的初始值是IV需要注意的是。 此外，每轮处理过程还需要计算常数表T中的16个子元素，分别是T[1..16]，T[17..32]，T[33..48]，T[49..64]。常数表的数据生成T[i]为x abs(sin(i))的整数部分，其中i是弧度单位，由于0 < abs（sin(i)) < 1，所以T[i]可以由32位表示。第4轮的输出需要再与第一轮输入的相加，相加是可以将看作4个32比特的子，每个字与第4轮输出对应的字按mod相加，相加的结果即为输出，即输出：（A、B、C、D)4个寄存器的当前值，即128位的，最终输出值为128位消息摘要。

四、MD5的安全性

​ MD5的安全性主要取决于压缩函数，128位的MD5算法；寻找具有相同哈希值的两个输入（类似第二类生日攻击）的攻击方法的复杂度为；给出一个特定的输入，找出具有相同哈希值的输入的攻击方法（类似第一类生日攻击）复杂度为；然而上述的攻击手段只是基于暴力破解。2004年，山东大学的王小云找出了MD5的碰撞，但只是属于破解强碰撞，还不算真正意义的破解，强碰撞就是找到任意2个具有相同哈希值的输入；并不是给出任意一个输入x，都能找出y，使得H(x)=H(y)且x！=y。王小云采用的差分攻击方法；此外针对哈希函数的攻击方法还有：哈希链表和彩虹表法、扩展长度攻击。
