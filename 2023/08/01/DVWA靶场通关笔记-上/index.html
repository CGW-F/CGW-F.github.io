Brute Force（暴力破解）

工具：Burp Suite pro、爆破字典、代理：搜狐浏览器+FoxyProxy标准版

【漏洞介绍】暴力破解是一种尝试在不知道密码的情况下，使用各种可能的密码组合来尝试访问系统的攻击方法。攻击者通常使用计算机程序来自动化执行这个过程，以尝试尽可能多的密码组合。这种攻击方法常常用于破解弱密码，例如常见的单词、数字或者简单的组合密码。

【防御】

  1. 强密码策略：实施强密码策略可以有效地防止暴力破解攻击。建议密码长度至少12个字符，包含大小写字母、数字和特殊字符。
  2. 账户锁定：设置账户锁定策略可以有效地防止暴力破解攻击。例如，在多次失败的登录尝试之后锁定用户账户，只有管理员可以解锁。
  3. 暂时延迟：为了防止攻击者快速地执行暴力破解，可以在失败的登录尝试之间引入延迟。这可以防止攻击者进行高速尝试。
  4. 双因素身份验证：使用双因素身份验证可以增加安全性，使得攻击者更难破解密码。例如，使用令牌、指纹识别等。
  5. IP限制：限制IP地址可以防止攻击者使用不同的计算机和网络进行尝试。例如，只允许特定的IP地址进行登录。
  6. 安全日志：记录安全日志可以帮助检测和识别暴力破解攻击。例如，记录登录尝试、失败的登录尝试等。

总之，防御暴力破解攻击需要综合使用多种技术和最佳实践，包括强密码策略、账户锁定、暂时延迟、双因素身份验证、IP限制和安全日志等。同时，还需要定期更新密码、使用不同的密码等。

Command Injection（命令注入）

【漏洞介绍】命令注入就是在需要输入数据的地方输入了恶意代码，而且系统并没有对其进行过滤或者其他处理导致恶意代码也被执行，最终导致数据泄露或者正常数据被破坏。有些应用中，可能允许操作者向后台操作系统发送命令并执行，例如测试同网站的连接（ping命令），这种功能一般通过如PHP中的exec、shell_exec和ASP中的wscript.shell等函数实现。命令注入漏洞指的是在这些功能中，由于对操作者的输入或返回的输出结果控制不严导致能够让恶意操作者利用管道符等将恶意系统命令拼接在正常命令后并执行。

【简单的指令】

Ipconfig：查看ip等信息

net user：查看系统用户

dir：查看当前目录

find;查找包含指定字符的行

whoami：查看系统当前有效用户名

systeminfo：查看系统信息

echo：输出打印

【常见命令拼接符】

  • A|B 不管A命令成功与否，都会去执行b命令。
  • &：A&B 各自执行
  • ||：A||B 先执行A，如果为假，再执行B
  • && ：A&&B 先执行A，如果为真，再执行B
  • ；： A;B 先执行A后执行B

File Inclusion（文件注入）

【漏洞介绍】文件注入漏洞（File Inclusion）是一种安全漏洞，允许攻击者在应用程序中引用和执行未经授权的文件。出现文件注入漏洞的原因通常是应用程序在动态引用文件时，没有正确地验证用户提供的输入，导致攻击者可以通过操纵这些输入来执行恶意代码。

【原理】文件注入漏洞发生在应用程序使用用户输入来动态生成文件路径的情况下。攻击者可以在用户输入中注入恶意代码，例如通过在文件路径中添加特殊字符或使用目录遍历攻击，来引用应用程序未经授权的文件。一旦攻击者成功执行了恶意代码，他们可以访问敏感信息，如配置文件、数据库凭据等，并且还可以在应用程序中执行任意代码。

【防御】为了防御文件注入漏洞，应用程序开发人员可以采取以下措施：

  1. 永远不要直接使用用户输入的数据来生成文件路径。而是使用白名单来限制允许访问的文件路径。例如，只允许访问特定的目录或文件。
  2. 对用户输入进行严格的输入验证和过滤，确保输入数据不包含特殊字符或目录遍历攻击。
  3. 使用安全编程语言，如PHP，提供内置函数来检查文件路径。例如，使用realpath()函数可以将文件路径规范化，从而防止目录遍历攻击。
  4. 避免在应用程序中使用可变文件名。如果必须使用可变文件名，确保在生成文件名之前对其进行过滤和验证。
  5. 对于包含敏感信息的文件，应该将它们放在应用程序根目录之外，并在操作系统层面上使用访问控制列表（ACL）限制文件的访问。

Insecure CAPTCHA

​ （不安全的验证码）

【常见的攻击方法】

  1. 自动化程序攻击：攻击者可以使用自动化程序来识别并绕过不安全的验证码，从而实现自动化攻击。
  2. 暴力破解攻击：攻击者可以使用暴力破解技术来破解验证码，从而实现攻击。例如，攻击者可以使用字典攻击或暴力破解工具来尝试所有可能的字符组合。
  3. 人工智能攻击：攻击者可以使用人工智能技术来分析和识别验证码，从而绕过验证码的限制。例如，攻击者可以使用深度学习算法训练模型来自动化地识别验证码
  4. 绕过技术

【防御】

  • 难度提升：提高验证码的难度可以有效地防止自动化程序和暴力破解攻击。例如，增加字符数量、添加背景噪音、旋转字符等。
  • 多种类型：使用多种类型的验证码可以增加攻击者绕过验证码的难度。例如，文字验证码、数学问题验证码、图像验证码等。
  • 时间限制：设置验证码的有效期限可以防止攻击者使用破解的验证码进行攻击。例如，验证码只在用户提交前几分钟内有效。
  • 验证码随机性：使用随机生成的验证码可以防止攻击者使用已知的验证码进行攻击。例如，使用随机的字符和字体。
  • 安全存储：验证码数据应该被安全地存储，以防止攻击者获取验证码数据并进行破解攻击。

总之，防御不安全的验证码需要综合使用多种技术和最佳实践，包括提高难度、使用多种类型、设置时间限制、使用随机性和安全存储等。
